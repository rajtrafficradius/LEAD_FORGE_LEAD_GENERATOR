# V5.8 Lead Generation Optimization - Smart Credit Conservation

## Overview

V5.8 implements intelligent lead generation optimization to dramatically reduce Apollo and SEMrush API credit consumption while maintaining data quality. When `maxLeads` is set (e.g., 20), the system now processes only the most relevant leads instead of searching through 700-800 candidates.

## Problem Statement

**Original Behavior (V5.7 and earlier):**
- Search for ~150 domains regardless of `maxLeads` setting
- Apollo returns 10-25 people per domain (~1500-3750 total people)
- Fetch complete data (name, email, phone) for ALL people
- Apply enrichment APIs to ALL people (Apollo + Lusha calls multiply the waste)
- Filter to top N only at CSV export stage

**Credit Waste:**
- For `maxLeads=20`: Fetched ~1800 people, used ~1500+ API calls, exported only 20
- Credit-per-lead ratio: 75+ credits per delivered lead (wasteful)

## Solution: V5.8 Smart Filtering

### Three-Layer Optimization

#### 1. Smart Domain Discovery (Phase 3)
- **Concept**: Reduce domains searched based on `maxLeads`
- **Implementation**: Domain cap = `max(30, maxLeads * 3)`
- **Example**: `maxLeads=20` → search 60 domains instead of 150
- **Savings**: 60% reduction in domain discovery API calls (SEMrush)

```python
if max_leads > 0:
    optimal_domain_cap = max(30, int(max_leads * 3))
else:
    optimal_domain_cap = 150  # Default unlimited behavior
```

#### 2. Lead Relevance Scoring
- **Concept**: Identify high-value leads by job title
- **Decision-Maker Keywords** (high relevance): CEO, CFO, Director, VP, Manager, President, Head of...
- **Low-Relevance Keywords** (skip enrichment): Intern, Assistant, Support, Coordinator, Technician...
- **Scoring Function**: `_calculate_lead_relevance_score(title)` returns 0-100

```python
DECISION_MAKER_KEYWORDS = {
    "ceo", "cfo", "director", "vp", "manager", "head",
    "chief", "founder", "owner", "president", ...
}

LOW_RELEVANCE_KEYWORDS = {
    "intern", "assistant", "support", "coordinator",
    "technician", "junior", ...
}
```

#### 3. Selective Enrichment (Phase 4)
- **Concept**: Skip expensive API calls for low-relevance people
- **Skipped for low-relevance**:
  - Step 2b: Apollo `enrich_person()` call
  - Step 2c: Apollo LinkedIn-based enrichment
  - Step 3: Lusha company info (if no high-relevance leads)
  - Step 4: Lusha `enrich_person()` call
- **Kept for all**: Initial Apollo people search, name/email extraction

```python
# In enrichment steps, check before making API call:
if max_leads > 0:
    role = lead.get("role", "").lower()
    is_low_relevance = any(kw in role for kw in LOW_RELEVANCE_KEYWORDS)
    if is_low_relevance:
        continue  # Skip expensive enrichment
```
